API

The API Tab, within the Settings page, gives details and access to the API.

The page is divided into two sections

  • Token and BaseURL
  • Vue API Documentation
    • The API documentation uses Swagger

Token and BaseURL

The Token and BaseURL section provides the following information

  • Token
  • BaseURL

  • Token

    • The Token is used to authenticate the API requests.

The token is specific to the tenant and great care should be taken to ensure that it is not leaked outside of the tenant company, as operations could be performed without the tenants authorisation.

  • BaseURL

    • This is the URL that all API calls are constructed on.
      • Example: https://staging-demo.safetyshieldvue.com/api/vehicles

Vue API Documentation

The endpoints are defined in the DEFAULT section

and each endpoint, when expanded, gives details on the request, such as payload

and response

The schema for each request is available in the schema section

Using the Swagger API UI

Step 1 - Get Authorisation

  • Copy the token

  • Open the Authorise modal by clicking on the Authorise button

  • Paste the token into the Bearer Token value box

  • Click the Authorize button

  • Close the Authorise modal

Step 2 - update the Host with the tenant baseURL

  • Copy the BaseURL, excluding the https://

  • Remove any data in the Host box

  • Paste the BaseURL into the Host box

  • Resulting in the following - but with the tenant-specific BaseURL

Step 3 - Make a request

  • Select an endpoint, from the DEFAULT section

  • Click on the TRY IT OUT button

  • Alter any parameters that the request may require (this example is a simple GET, with no parameters) and press the EXECUTE button

  • The request will be made

  • The response will be displayed

  • The response may include multiple data sets, which you can see by scrolling the response section
